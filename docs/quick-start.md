# 快速开始

CodePet V0.4.2 在保留 V0.1 桌宠、V0.2 提醒、V0.3 Ollama 本地 AI、V0.3.5 轻量首页和 V0.4.1 通用命令监控的前提下，新增 Codex / Cursor / Claude Code CLI Adapter 与用户接管提醒增强。

## 启动开发版

```bash
cd apps/desktop
pnpm install
pnpm tauri dev
```

启动后你会看到 CodePet 本地工作面板。窗口可拖动、可置顶，关闭后会隐藏到系统托盘。

## 使用首页

默认首页包含以下区域：

1. **顶部问候区**：根据时间显示早安 / 下午好 / 晚上好，右上角展示桌宠在线状态与 Ollama 连接状态。
2. **今日提醒**：展示最多 3 条已启用提醒。
3. **自定义角色**：展示 4 个轻量角色预设卡片。
4. **最近任务状态**：展示最近 3 条命令任务摘要，支持运行中、已完成、失败、等待确认、长时间无输出、已取消等状态。

## 创建并运行命令任务

### 通用命令

1. 点击左侧导航「任务监控」。
2. 点击「新建命令任务」，选择任务类型「通用命令」。
3. 填写标题、工作目录（可点击「选择目录」）、执行命令（如 `pnpm`）和参数（如 `build`）。
4. 点击「创建任务」后，在右侧选择任务并点击「启动任务」。
5. CodePet 会要求你确认启动，不会对危险命令自动执行。
6. 运行中可展开查看 stdout / stderr 日志；完成后桌宠会显示成功或失败状态。

### Agent CLI 任务（Codex / Cursor / Claude Code）

1. 在任务监控页点击「依赖检测」，配置各 CLI 可执行路径并重新检测。
2. 新建任务时选择 Codex / Cursor CLI / Claude Code。
3. 填写工作目录、prompt / 命令内容，可选设置无输出超时分钟数（默认 5）。
4. 由你主动点击启动；CodePet 不会自动确认权限或自动执行 AI 建议的命令。
5. 若输出中出现等待确认关键词，任务会变为「等待确认」并触发桌宠提醒。
6. 若长时间无新输出，任务会变为「长时间无输出」，你可选择继续等待或取消任务。

示例：

| 标题 | 工作目录 | 命令 | 参数 |
|------|----------|------|------|
| 桌面端构建 | `E:/code/codepet/apps/desktop` | `pnpm` | `build` |
| 桌面端测试 | `E:/code/codepet/apps/desktop` | `pnpm` | `test` |

## 进入提醒管理

1. 点击左侧导航「提醒」，或首页「今日提醒」区的「查看全部」。
2. 填写标题、类型、模式和文案。
3. 选择提示音并创建提醒。

## 使用本地 AI

1. 点击左侧导航「本地 AI」。
2. 在「AI 设置」中检测 Ollama、选择模型并启用本地 AI。
3. 切换到「本地聊天」测试模型回复。

## 设置与桌宠调试

在「设置」页可以切换窗口置顶，并使用桌宠状态测试面板手动切换状态。

## 当前不要期待的功能

- 没有 Petdex 素材导入（V0.4.5）。
- 没有宠物孵化提示词向导（V0.4.6）。
- 没有自动确认 Agent 权限、自动修复代码或自动 git commit / push。
- 没有记忆、Skill 执行或多角色圆桌会议。
