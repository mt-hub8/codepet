# Skill 系统

Skill（技能）表示可配置的能力。早期 Skill 可以是配置驱动，而不是代码插件。

## Skill 可以包含什么

- 触发条件。
- Prompt（提示词）模板。
- 可用工具。
- 权限说明。
- 适用角色。

## 权限原则

Skill 不应该默认获得全部权限。每个 Skill 的能力边界必须明确，例如是否能读取任务日志、是否能调用外部命令、是否能访问本地文件。

## 用户确认

后续可以支持根据用户习惯建议 Skill，但自动生成的 Skill 必须用户确认后才能启用。

## 后续能力

- 支持用户导入 Skill。
- 支持 Skill 模板。
- 支持角色绑定 Skill。
- 支持根据使用习惯建议 Skill。

V0.0 不实现真实 Skill 执行。
