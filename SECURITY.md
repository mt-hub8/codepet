# 安全政策

## 如何报告安全问题

如果你发现 CodePet 的安全漏洞，**请不要**在公开 Issue 中披露细节（避免 API Key、Token、完整日志或可利用的 PoC 公开传播）。

请通过以下方式联系维护者：

1. 在 GitHub 使用 **Private vulnerability reporting**（若仓库已启用），或
2. 向仓库维护者发送私信（Security Advisory），包含：
   - 问题描述与影响范围
   - 复现步骤（可脱敏）
   - CodePet 版本与操作系统
   - 是否已有缓解措施

我们会在合理时间内确认并回复。

## 当前安全边界

CodePet 的设计目标是**本地优先**：

- 默认不上传用户代码、命令日志、聊天、提醒、宠物素材
- 不内置云 API Key，不处理外部工具登录
- 不做应用内遥测
- 安装包不包含用户运行期数据

## 命令执行风险

命令监控功能会**在用户主动确认后**在本机执行命令：

- 对明显危险命令（如 `rm -rf`、`del /s`、`format`）会二次确认
- CodePet **不会**自动修改你的代码或重新执行失败命令
- 请仅在信任的工作目录中创建任务
- Windows 路径与权限可能导致命令行为与预期不符

**不要在 Issue 中粘贴包含密钥或私密路径的完整命令日志。**

## 外部 Agent CLI 风险

Codex / Cursor / Claude Code 等工具可能：

- 请求文件系统或网络权限
- 将代码或日志发送到各自的后端（取决于工具配置）

CodePet 仅监控输出并提醒，**不会**替用户确认权限或自动输入。请阅读各工具的安全与隐私文档。

## 宠物素材版权与来源

用户导入的 `pet.json` 与 spritesheet 可能涉及第三方版权。CodePet 不对用户导入素材的合法性负责。请勿在仓库或 Issue 中上传无授权素材。

## 隐私边界

- 诊断信息复制时已做路径脱敏，但仍可能包含模型名、应用版本等
- 远程 Ollama 地址会将聊天内容发往该服务
- 卸载应用后本地数据可能仍保留，需手动删除数据目录

详见 [docs/privacy.md](docs/privacy.md)。

## Issue 中请勿粘贴

- API Key、Token、密码
- 未脱敏的完整命令日志
- 私人项目路径或仓库 URL（若含敏感信息）
- 他人无授权的宠物素材原图

## 代码签名

当前 **Public Beta 安装包未代码签名**。Windows 可能提示未知发布者；部分杀毒软件可能误报。请仅从 [GitHub Releases](https://github.com/mt-hub8/codepet/releases) 下载。

## 披露时间线（目标）

- 确认收到：7 个工作日内
- 修复或缓解计划：视严重程度协商
- 公开披露：修复发布后（或与维护者协商）

感谢帮助 CodePet 更安全。
